# captionWorkingWithAI
AI로 작업하는 캡컷 자막 편집

## 캡컷이 만든 자동자막 srt파일을 제미나이에게 수정 요청
아, 정말 똑똑한 작업 방식입니다! 캡컷의 자동 자막을 `.srt`로 내보내서 AI에게 교정을 맡기면 작업 속도가 비약적으로 빨라질 겁니다.

저(Gemini)에게 작업을 시킬 때, **가장 정확한 결과물을 얻을 수 있는 '명령어(프롬프트)'**를 만들어 드립니다. 아래 내용을 복사해서 `.srt` 파일 내용과 함께 붙여넣으시면 됩니다.

---

### 📥 제미나이 입력용 프롬프트 (복사해서 쓰세요)

아래 내용을 복사해서 저에게 주실 때, **맨 아래에 srt 파일의 텍스트 내용을 붙여넣어 주세요.**

```markdown
당신은 '프로 영상 편집자'입니다. 아래 제공된 자막 파일(.srt)의 내용을 한국어 문법과 가독성 규칙에 맞춰 교정해 주세요. 타임스탬프와 순서 번호는 절대 건드리지 말고, 오직 '텍스트'만 수정해야 합니다.

**[수정 규칙]**

1. **줄바꿈 (Semantic Line Breaks)**
   - 문장의 의미 단위로 줄을 나누세요.
   - 조사(은/는/이/가/을/를 등)가 명사와 떨어져서 다음 줄로 넘어가지 않게 하세요. (예: '밥을'은 붙여 쓰고, '밥'과 '을'을 줄바꿈으로 나누지 말 것)
   - 어미가 끊기지 않게 하세요.

2. **길이 제한 (Length Constraints)**
   - 한 화면(타임스탬프 구간) 당 자막은 **최대 2줄**을 넘지 않도록 합니다.
   - 한 줄의 길이는 **공백 포함 약 15~20자** 내외로 맞춰 시청자가 한눈에 읽을 수 있게 하세요.
   - 2줄로 나눌 때는 윗줄보다 아랫줄이 약간 더 길거나 비슷하게(피라미드 구조) 배치하여 시각적 안정감을 주세요.

3. **맞춤법 및 띄어쓰기**
   - 기본적인 오타와 띄어쓰기를 교정하세요.
   - 구어체 느낌은 살리되, 가독성을 해치는 지나친 구어체 표기는 표준어에 가깝게 다듬으세요.

4. **출력 형식**
   - 수정된 내용을 복사하기 쉽게 **코드 블록(Code Block)** 안에 .srt 형식 그대로 출력해 주세요.

**[여기부터 자막 내용입니다]**
(여기에 srt 파일 내용을 붙여넣으세요)

```

### 💡 이 프롬프트의 핵심 원리

제가 위 규칙을 어떻게 이해하고 처리하는지 설명해 드릴게요.

1. **조사 분리 금지:** 한국어 자막에서 가독성을 망치는 주범이 바로 "나는 밥 / 을 먹었다" 처럼 조사가 혼자 떨어지는 것입니다. 이 규칙을 넣으면 **"나는 밥을 / 먹었다"**로 깔끔하게 교정합니다.
2. **피라미드 구조:** 사람의 눈은 역삼각형(▽)보다 삼각형(△) 구조의 텍스트를 더 안정적으로 느낍니다. 2줄일 때 아랫줄을 조금 더 길게 잡아달라고 요청했습니다.
3. **코드 블록 출력:** 캡컷은 `.srt` 형식을 그대로 불러와야 인식합니다. 제가 그냥 줄글로 답하면 다시 포맷팅해야 하므로, **복사 버튼 하나로 끝낼 수 있게** 코드 블록 요청을 넣었습니다.



## 가독성 좋은 자막의 4가지 황금 규칙

캡컷의 자동 자막 기능이 정말 편리하긴 하지만, 말씀하신 대로 한국어의 띄어쓰기나 줄바꿈이 어색해서 그대로 쓰기에는 무리가 있죠. 시청자가 영상을 보며 **'뇌를 쓰지 않고도'** 이해할 수 있도록 만드는 **가독성 좋은 자막의 4가지 황금 규칙**을 정리해 드립니다.

이 규칙대로만 수정하면 훨씬 전문적인 영상처럼 보일 것입니다.

---

### 1. 줄바꿈과 길이 (가장 중요)

자동 자막이 가장 못하는 부분입니다. 시청자는 글자를 읽는 게 아니라 **이미지처럼 인식**해야 합니다.

* **의미 단위로 끊기:** 문법보다는 '말하는 호흡'이나 '의미 덩어리'로 줄을 바꿔야 합니다. 조사가 혼자 덜렁 떨어지지 않게 주의하세요.
* *(X) 오늘은 날씨가 정말 / 좋아서 밖으로 나갔다.*
* *(O) 오늘은 날씨가 정말 좋아서 / 밖으로 나갔다.*


* **두 줄 법칙:** 한 화면에 자막은 **최대 두 줄**을 넘기지 마세요. 세 줄이 넘어가면 영상 화면을 가리고 시선을 분산시킵니다.
* **피라미드 구조 (권장):** 두 줄일 경우, 윗줄보다 아랫줄이 긴 것이 시각적으로 안정적입니다.
* *(윗줄 짧게)* 맛있는 음식을
* *(아랫줄 길게)* 먹으러 가볼까요?



### 2. 글꼴(Font)과 디자인

화려한 것보다 **잘 보이는 것**이 우선입니다.

* **고딕 계열 사용:** 영상 자막은 삐침이 있는 명조체(Serif)보다 획이 굵고 깔끔한 **고딕체(Sans-serif)**가 압도적으로 유리합니다. (예: G마켓 산스, 프리텐다드, 여기어때 잘난체 등)
* **테두리와 그림자:** 영상 배경은 계속 바뀝니다. 흰색 글씨만 쓰면 밝은 배경에서 안 보입니다.
* **검은색 테두리(Stroke)**를 씌우거나,
* **반투명 검은색 배경(Background)**을 깔아주는 것이 국룰입니다.


* **색상 제한:** 자막의 색상은 2~3가지를 넘기지 마세요.
* 기본: 흰색 + 검은 테두리
* 강조: 노란색(또는 형광색) + 검은 테두리



### 3. 위치 선정 (세이프 존)

자막을 어디에 두느냐에 따라 영상의 퀄리티가 달라집니다.

* **숏폼(쇼츠, 릴스)의 경우:** 하단에 설명글, 좋아요 버튼, 아이디 등이 뜹니다. 자막이 이 UI들에 가려지지 않도록 **생각보다 꽤 위쪽**에 배치해야 합니다.
* **시선 처리:** 화자의 얼굴과 너무 멀어지면 안 됩니다. 보통 **화면 하단 1/3 지점**이 가장 안정적입니다.

### 4. 캡컷(CapCut) 편집 꿀팁

캡컷 기능을 활용해 빠르게 수정하는 방법입니다.

* **일괄 편집 (Batch Edit):** 타임라인에서 하나씩 수정하지 말고, 자막을 선택 후 **[일괄 편집]**을 누르세요. 텍스트 문서처럼 쭉 이어서 보며 엔터(Enter) 키로 줄바꿈을 빠르게 수정할 수 있습니다.
* **템플릿 활용:** 캡컷의 '텍스트 템플릿' 중 예능 자막 스타일을 고른 뒤, 폰트만 깔끔한 것으로 바꿔서 '스타일 복사'하여 전체 적용하면 시간을 획기적으로 줄일 수 있습니다.
